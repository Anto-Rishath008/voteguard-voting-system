Current Problem: You're getting "403 Forbidden - Admin or SuperAdmin access required" even though you're logged in.

Why This Is Happening

Your account exists and you can log in, but you don't have the Admin role in the database. The system creates accounts with the default "Voter" role. Admin roles must be manually assigned.

The Solution (5 Minutes)

Step 1: Open Supabase SQL Editor

  1. Go to Open Supabase Dashboard
  2. Sign in to your account
  3. Select your voting system project
  4. Click "SQL Editor" in the left sidebar
  5. Click "New Query" button

Step 2: Find Your User

Copy and paste this SQL into the editor, then click Run:

SELECT u.user_id, u.email, u.first_name || ' ' || u.last_name as name, COALESCE(array_agg(ur.role_name), ARRAY[]::text[]) as current_roles FROM users u LEFT JOIN user_roles ur ON u.user_id = ur.user_id GROUP BY u.user_id, u.email, u.first_name, u.last_name ORDER BY u.created_at;

Look for YOUR email in the results and check the current_roles column. It probably shows {} (empty) or {Voter}.

Step 3: Add Admin Role

Option A - Give admin to first user (easiest):

DO $$ DECLARE v_user_id uuid; v_email text; BEGIN SELECT user_id, email INTO v_user_id, v_email FROM users ORDER BY created_at ASC LIMIT 1; INSERT INTO user_roles (user_id, role_name, assigned_at) VALUES (v_user_id, 'Admin', CURRENT_TIMESTAMP) ON CONFLICT (user_id, role_name) DO NOTHING; RAISE NOTICE '‚úÖ Admin role added to: %', v_email; END $$;

Option B - Use your specific email:

DO $$ DECLARE v_user_id uuid; v_email text := 'YOUR-EMAIL@EXAMPLE.COM'; -- ‚ö†Ô∏è CHANGE THIS! BEGIN SELECT user_id INTO v_user_id FROM users WHERE email = v_email; INSERT INTO user_roles (user_id, role_name, assigned_at) VALUES (v_user_id, 'Admin', CURRENT_TIMESTAMP) ON CONFLICT (user_id, role_name) DO NOTHING; RAISE NOTICE '‚úÖ Admin role added!'; END $$;

Remember to replace YOUR-EMAIL@EXAMPLE.COM with your actual email!

Step 4: Verify It Worked

Run this query (replace the email):

SELECT email, array_agg(ur.role_name) as roles FROM users u LEFT JOIN user_roles ur ON u.user_id = ur.user_id WHERE u.email = 'YOUR-EMAIL@EXAMPLE.COM' GROUP BY email;

You should see {Admin} in the roles column.

‚ö†Ô∏è CRITICAL STEP - Don't Skip!

You MUST log out and log back in for this to take effect!

Your current login session has a JWT token that was created BEFORE you had the Admin role. The token only updates when you log in.

Step 5: Log Out and Back In

  • Go to http://localhost:8000
  • Click "Log Out"
  • Log back in with your email and password
  • The system will create a new JWT token with your Admin role

After logging back in, visit:

Open Admin Users Page

You should see the admin panel with no 403 error!
